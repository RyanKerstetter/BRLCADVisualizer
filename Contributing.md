## Creating Good Issues

When submitting an issue, please include:

- A clear and descriptive title  
- A detailed description of the problem  
- Steps to reproduce the issue  
- Expected behavior versus actual behavior  
- Screenshots or logs (if applicable)  
- Environment details (OS, GPU, build configuration)

### Example

Bad:
"Rendering broken"

Good:
"PathTracer produces black output after switching from AO renderer"

---

## Creating Good Pull Requests

To keep contributions easy to review:

- Keep pull requests small and focused  
- Provide a clear description of what changed and why  
- Link related issues (e.g., Closes #12)  
- Include before/after screenshots for UI or rendering changes  
- Ensure all tests pass before submitting  

### Pull Request Checklist

- Code builds successfully  
- Tests pass  
- No unnecessary files included  
- Changes are documented if needed

## Code of Conduct

All contributors are expected to:

- Be respectful and professional  
- Communicate clearly and constructively  
- Accept feedback and collaborate effectively  
- Avoid hostile, dismissive, or disruptive behavior  

The guiding principle is simple:
Treat others as teammates working toward a shared goal.

---

## Community Expectations

- Ask questions if something is unclear  
- Share knowledge when possible  
- Prioritize clarity over complexity in code  
- Focus on improving the project rather than individual preferences  

Contributions should aim to:

- Improve usability  
- Maintain stability  
- Enhance performance or clarity
